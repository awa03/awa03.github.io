Dynamic Memory Allocation

Static (compile time)- Fixed sizes and types of memory, including in arrays. Space must be given to data. Static arrays are not able to grow or shrink.
Dynamic (run time)- Memory which is allocated at run time, it is not fixed. The dynamic memory doesnt have a name known by the compiler so pointers are used for this type of memory

Syntax

Allocate dynamic space with operator new, which returns address of the allocated item. Store in a pointer:
int * ptr = new int; // one dynamic integer
double * nums = new double[size]; // array of doubles, called "nums"
Clean up memory with operator delete. Apply to the pointer. Use delete [] form for arrays:
delete ptr; // deallocates the integer allocated above
delete [] nums; // deallocates the double array above
Remember that to access a single dynamic item, dereference is needed:
cout << ptr; // prints the pointer contents
cout << *ptr; // prints the target
For a dynamically created array, the pointer attaches to the starting position of the array, so can act as the array name:
nums[5] = 10.6;
cout << nums[3];

Dynamic Allocation of Objects

Objects can also be dynamically allocated. Deallocation works the same as if it were within the member functions. We need to deallocate the memory that we have allocated.For dynamically allocated arrays of objects, the pointer acts as the array name, but the object "names" can be reached with the bracket operator. Arrow operator usually not needed:

Dot Operator or Arrow Operator?

The dot operator and the arrow operator have a similar structure (calling the member functions). Pointers must be derefrenced before calling the pointed object, however the arrow operator may be used in place of this.

Using DMA inside a class

To start creating an array of objects that is dynamically allocated we need to setup dynamic memory as part of out member data, we must have the pointers initalized within the function. Using the new keyword we can allocate space attaching it to the pointers. Make sure and delete the allocated space when done. It is best practice to seperate memory allocation tasks from algorithmic ones.

Entry -- An object of this class type represents a single entry in a phone book. The data members stored in an entry object are name, address, and phone number. Strings (i.e. null-terminated character arrays) are used to store these items. Directory -- An object of type Directory stores a list of Entry objects, using a dynamic array. The Directory class also provides services (public member functions) for adding new entries, deleting entries, modifying entries, searching for entries, and displaying all entries in the phone book. The Directory class also has a function for dynamically resizing the array of Entries when more memory space is needed. The destructor is implemented within the directory.
