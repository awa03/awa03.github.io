Intro To Data Structures

C++ has built in methods of storing compount data in many ways. Some examples include arrays and structs.

The most basic data structure is an array. An array is a collection of data of the same type. The general concept of what a data container stores, along with gerneral operation that want that data, is an example of an abstract data type.

A sequence is a data type that stores data in a linear order. For example, a list of names would be written in a specific order, and may have duplicates.
A set is a data type that stores data in a non-linear order. For example, a venn diagram is a set of data that is not in a linear order. In the set of all students wearing hats, you may be wearing a hat, you may not.

The study of data structures is the study of how to store data in a way that is efficient for the program. There are advantages of each method, and with that disadvantages. Common data structures include: stacks, queues, vectors, linked lists, trees, hash tables, sets, and others.

Stacks

Stacks are abstract data types. They are a collection of data that stores data in a last in first out order. This means that the last item added to the stack is the first item removed from the stack. An example being a stack of plates. The last plate added to the stack is the first plate removed from the stack.

The stack class has two primary operations.
push -- adds an item onto the top of the stack
pop -- removes the top item from the stack

The stack class also has two secondary operations. These operations are not required for a stack, but are useful. Typical application areas are within the compiler, operating system, handling of program memory.

Queues

Queues are abstract data types. They are a collection of data that stores data in a first in first out order. This means that the first item added to the queue is the first item removed from the queue. An example being a line at a grocery store. The first person in line is the first person to be checked out.

The queue class has two primary operations.
enqueue -- adds an item onto the back of the queue
dequeue -- removes the front item from the queue

The queue class also has two secondary operations. These operations are not required for a queue, but are useful. Typical application areas are within print job scheduling, operating system scheduling.

Vectors

Vectors are specific implementation of the sequence/list ADT. They are a collection of data that stores data in a linear order. This means that the data is stored in a linear order, and can be accessed by index. An example being a list of names. The list of names is stored in a linear order, and can be accessed by index.

The vector class has two primary operations.
push_back -- adds an item onto the back of the vector
pop_back -- removes the last item from the vector

The vector class also has two secondary operations. These operations are not required for a vector, but are useful. Typical application areas are within the compiler, operating system, handling of program memory.

Vectors are made by encapsulating an array within a class (vector class).
This comes with the advantages of Random access, and dynamic memory allocation.
However, this comes with the disadvantage of having to allocate memory for the vector. Which insert and delete functions are typically slow, and may require complex shifting of data.

Linked Lists

This is a specific implementation of the sequence/list ADT. They are a collection of data that stores data in a linear order. It stores a collection of adt items linked together with pointers, "lined up in a row". Made up of a collection of self referential nodes. Which are created from a self-referential class. This is also known as a struct.

Self-referential class: is a class who's member data contains at least one pointer to an object of the same class type. Each node contains a piece of data, and a pointer to the next node.
Nodes can be anywhere in memory, and are linked together by pointers.
Nodes generally allocated dynamically, and are linked together by pointers. So they can grow to any size.

This is an alternative to array based storage
Advantages: Dynamic memory allocation, Insertion and deletion is relativly fast. Require only the creation of a new node and the changing of a few pointers.
Disadvantages: Random access is not possible. Must traverse the list to find the desired node. Possible to build indexing into a linked list. However, this is not a standard feature.
The avantages of the array (vector) are generally disadvantages of the linked list vice versa.
