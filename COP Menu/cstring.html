C Strings

C-strings are implemented as null-terminated arrays of type char There is no built in string class within c++, so if we dont want to use the "#include string" we need to make a cstring!

Library Uses/Features

The c string library contains some common functions for string operations. the examples are shown below:

strlen()
strcpy(), strncpy()
strcat(), strncat()
strcmp(), strncmp()
strstr()
strtok()
etc.

The Negatives of C-Strings

C-strings are pretty efficient, since they are just character arrays. If you need a fixed size string that works fast, this is a great option. But the user of the c-string has the responsibility for calling the library functions correctly. They are not foolproof!

It may be beneficial to create your own string library because of the limitations of c-strings. C-strings are fixed in size, when declared as a normal c style array. The c-string name also acts as a pointer because it is only the name of the array and not the value within. This is why it must always be passed by address within functions. Overflow may easily happen as well, because of this it can be risky to use c-strings in larger databases.

The Null Character

C-Strings largest pitfall is that they are based on the expectation of a null char to end the loop This may lead to a few potential errors.

An example of this being copying a string to a out of bounds array:

char buffer[40] = "Dog"; // length 3, capacity 39
char word2[] = "food"; // length 4, capacity 4

strcat(buffer, word2); // buffer is now "Dogfood"
strcat(buffer, " breath"); // buffer is now "Dogfood breath"
strcat(buffer, buffer);

Building A String Class

Dynamic memory allocation makes building a dynamic string class possible. Operator operloads allow the string class to be more useful.
